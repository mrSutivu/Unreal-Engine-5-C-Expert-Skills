# Class Prefixes (UE5 Expert)

## Activation / When to Use
- Mandatory for all new class, struct, and enum declarations.
- Trigger when creating new C++ files or refactoring existing ones.
- Essential for ensuring reflection data is correctly generated by UHT.

## 1. The Prefix Matrix
Unreal Engine uses specific prefixes to categorize types for its reflection and memory management systems.

| Prefix | Category | Description |
| :--- | :--- | :--- |
| **A** | **Actor** | Classes inheriting from `AActor`. Can be spawned in a level. |
| **U** | **Object** | Classes inheriting from `UObject`. Cannot be spawned in a level alone. |
| **F** | **Plain Data/Struct** | `USTRUCT()` or native C++ classes/structs. |
| **E** | **Enum** | All `UENUM()` declarations. |
| **I** | **Interface** | Classes inheriting from `UInterface`. |
| **T** | **Template** | Template classes (e.g., `TArray`, `TMap`). |
| **b** | **Boolean** | All boolean member variables (e.g., `bIsActive`). |

## 2. Implementation Rules

### Correct Declaration
The prefix must be present in the C++ declaration, but IS NOT part of the filename.
- **Filename**: `MyActor.h`
- **Class**: `class AMyActor : public AActor`

```cpp
// CORRECT
UCLASS()
class MYPROJECT_API AMyWeapon : public AActor { ... };

USTRUCT(BlueprintType)
struct FWeaponStats { ... };

UENUM(BlueprintType)
enum class EWeaponState : uint8 { ... };
```

### Reflection System Compatibility
If you omit the prefix (e.g., `class MyObject`), the Unreal Header Tool (UHT) will fail to generate the necessary `.generated.h` data or throw a compilation error.

## 3. Impact on Safety
- **UHT/Linker Integrity**: Naming consistency prevents "Mystery Crashes" caused by incorrect reflection data.
- **Serialization**: Ensures the engine can properly serialize data for SaveGames and Networking.
- **Code Clarity**: Allows developers to immediately identify the memory management rules of a type just by its name.

## Common Mistakes (BAD vs GOOD)

**BAD (Missing Prefix)**:
```cpp
UCLASS()
class PlayerManager : public UObject { ... }; // UHT ERROR
```

**GOOD (Correct Prefix)**:
```cpp
UCLASS()
class UPlayerManager : public UObject { ... }; // OK
```

**BAD (Wrong Prefix for Type)**:
```cpp
struct AMyData { ... }; // MISLEADING: 'A' is for Actors only. Use 'F'.
```

## Verification Checklist
- [ ] Actors start with `A`.
- [ ] UObjects (Components, Subsystems) start with `U`.
- [ ] Structs start with `F`.
- [ ] Enums start with `E`.
- [ ] Booleans start with `b`.
- [ ] Filenames do NOT include the prefix.
