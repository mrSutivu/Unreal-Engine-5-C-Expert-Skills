# Shader Compilation & Debugging (UE5 Expert)

## Activation / When to Use
- Mandatory when developing or iterating on `.usf` / `.ush` files.
- Trigger when the engine crashes due to a GPU timeout (TDR) or an HLSL syntax error.

## 1. Enabling Shader Development Mode
By default, Unreal Engine hides shader compile errors and fails silently. You MUST enable Development Mode.

### Console Variables (CVars)
Add these to your `ConsoleVariables.ini` (located in `Engine/Config/` or `Project/Config/`) to enforce them globally:

```ini
; Exposes detailed HLSL syntax errors in the Output Log
r.ShaderDevelopmentMode=1

; Dumps the intermediate HLSL code generated by the engine to disk
r.DumpShaderDebugInfo=1

; Forces synchronous compilation (helpful for catching exact crash moments)
r.CompileShadersForDevelopment=1
```

## 2. Live Recompilation Workflow
When modifying `.usf` or `.ush` files on disk, you do NOT need to restart the Editor or recompile C++ (Live Coding).

1. Edit and save the `.usf` file in your IDE.
2. In the Unreal Editor, press **`Ctrl + Shift + .`** (Control + Shift + Period).
3. The engine will instantly recompile all global and material shaders that reference the changed file.

## 3. Investigating Dumped Shaders
When `r.DumpShaderDebugInfo=1` is active, the engine saves the fully pre-processed HLSL code (after all `#includes` and macros are expanded) to the project's `Saved/ShaderDebugInfo/` folder.

- **Why this is critical**: The code you write in a Custom Node or a USF file is wrapped in massive engine templates. If you get an error like "Line 3450: syntax error", you must open the dumped file in `ShaderDebugInfo` to see exactly what Line 3450 is.

## 4. Handling GPU Crashes (TDR)
A "Timeout Detection and Recovery" (TDR) crash means your Compute Shader or Pixel Shader took too long (infinite loop).

### CVar to increase TDR limit (Windows Registry)
Unreal provides a registry key bypass for heavy shader development, but temporarily via CVars:
```ini
r.D3D12.TDR=0 ; Sometimes delays the crash allowing debug dumps
```

## 5. Impact on Safety
- **Blind Iteration**: Without `r.ShaderDevelopmentMode=1`, agents/developers are flying blind, guessing why a post-process effect isn't rendering.
- **Engine Freezes**: Catching infinite loops `for(;;)` in HLSL is impossible without debugging tools, requiring hard reboots of the OS in worst-case DX12 crashes.

## Verification Checklist
- [ ] `r.ShaderDevelopmentMode=1` is active when modifying USF files.
- [ ] Live Recompile shortcut (`Ctrl+Shift+.`) is used to test changes.
- [ ] `Saved/ShaderDebugInfo` is checked when complex macro or include errors occur.
